## 🛠️ Important Architectural Decisions

**Decision:** Pydantic Structured Outputs
**Why:** The Checker Team identified raw JSON parsing as a high risk for LLM hallucination crashes. We migrated to `output_pydantic` in `agents.py` to enforce schema constraints.

**Decision:** Graceful Error Handling on Tools
**Why:** `ScrapeWebsiteTool` could crash the run if a site was protected by Cloudflare. Wrapped it in `SafeScrapeWebsiteTool` to return a friendly string on failure.

**Decision:** Human-in-the-Loop Dispatcher
**Why:** The engine generates drafts but needed a safe steering wheel to review and send them. `dispatcher.py` acts as the manual UI.

**Decision:** Container-First Infrastructure & PostgreSQL
**Why:** To ensure the system runs identically across all environments and to prepare for production deployment, we migrated from local execution to Docker (`compose.yaml`). We upgraded the database from SQLite to PostgreSQL 15 to match the production target database architecture.
